Перенос XML Parser v5 на сервер Битрикс24 (закрытый контур)
1. Что нужно понять заранее
XML Parser — это отдельное PHP-приложение. Оно не интегрируется внутрь Битрикс24, а просто живёт рядом на том же сервере (или другом). Битрикс24 тут — просто сервер с PHP, который ты используешь как площадку.

2. Какие доступы запросить
#	Доступ	Зачем	Как сформулировать запрос
1	SSH или RDP к серверу	Залить файлы, настроить	«Нужен доступ к серверу для размещения PHP-приложения»
2	Путь к веб-директории	Куда положить файлы	«В какую папку можно положить PHP-проект чтобы он был доступен по HTTP?»
3	PHP версия и расширения	Проверить совместимость	«Какая версия PHP? Включены ли ext-curl, ext-simplexml, ext-json, ext-mbstring?»
4	Сетевой доступ к 1С	Отправка заказов	«Нужен доступ с сервера Б24 на http://10.30.88.24:80 (или ваш IP 1С)»
5	Cron / Планировщик	Автоматический запуск	«Можно ли настроить cron-задачу для запуска PHP-скрипта каждые N минут?»
6	Права на запись	Логи, JSON, перемещение XML	«PHP-процессу нужны права на запись в папку проекта»
3. Пошаговый план переноса
Шаг 1. Узнать обстановку

Спросить у админа:
- Какая ОС? (Linux / Windows)
- Какая версия PHP?
- Где веб-корень? (/var/www/html/ или C:\inetpub\...)
- PHP работает через Apache, Nginx или IIS?
- Есть ли доступ к php.ini?
Шаг 2. Проверить PHP
Создать файл phpinfo.php на сервере:

php

<?php phpinfo(); ?>
Открыть в браузере и проверить:

✅ PHP ≥ 7.0
✅ curl — enabled
✅ simplexml — enabled
✅ json — enabled
✅ mbstring — enabled
Удалить phpinfo.php после проверки!

Шаг 3. Залить файлы

Вариант А: через SSH
  scp -r parser_v5/ user@server:/var/www/html/parser_v5/

Вариант Б: через FTP/SFTP
  Любым клиентом (FileZilla, WinSCP)

Вариант В: архивом
  1. Запаковать в zip
  2. Залить на сервер
  3. Распаковать
Структура на сервере:


/var/www/html/parser_v5/     ← или другой путь
├── config/
├── core/
├── parsers/

Шаг 4. Права на папки
bash

# Linux
chmod -R 755 /var/www/html/parser_v5/
chmod -R 777 /var/www/html/parser_v5/input/
chmod -R 777 /var/www/html/parser_v5/json/
chmod -R 777 /var/www/html/parser_v5/logs/
chmod -R 777 /var/www/html/parser_v5/config/

# Или через владельца (правильнее)
chown -R www-data:www-data /var/www/html/parser_v5/

Windows: убедиться что IIS_IUSRS имеет права на запись
в папки input/, json/, logs/, config/
Шаг 5. Настроить config/settings.json
json

{

Шаг 6. Проверить связь с 1С
bash

# С сервера Б24 выполнить:
curl -v http://10.30.88.24/TRADE/hs/CRM_Exchange/Order

# Если таймаут — сеть закрыта, нужно просить открыть
Шаг 7. Проверить через браузер

http://server-b24/parser_v5/index.php     — панель управления
http://server-b24/parser_v5/test.php      — тесты (должны пройти)
http://server-b24/parser_v5/data.php      — таблица (пустая пока)
Шаг 8. Настроить cron
bash

# Linux — добавить в crontab
crontab -e

# Каждые 5 минут
*/5 * * * * /usr/bin/php /var/www/html/parser_v5/process.php >> /dev/null 2>&1

Windows — Планировщик задач:
  Программа: C:\php\php.exe
  Аргументы: C:\inetpub\parser_v5\process.php
  Расписание: каждые 5 минут
4. Возможные проблемы
🔴 Критические (не запустится)
Проблема	Симптом	Решение
PHP < 7.0	Белый экран, syntax error	Обновить PHP (маловероятно на Б24)
ext-curl отключён	Fatal Error при отправке	Раскомментировать extension=curl в php.ini
Нет прав на запись	Ошибки при парсинге, пустые логи	chmod 777 или chown
Сеть до 1С закрыта	Таймаут при отправке, «API недоступен»	Запросить открытие порта
🟡 Средние (работает частично)
Проблема	Симптом	Решение
Битрикс перехватывает URL	404 при открытии parser_v5/	Настроить алиас в Nginx/Apache
.htaccess конфликт	Редирект на главную Б24	Добавить свой .htaccess в папку парсера
Таймаут PHP слишком мал	Обработка обрывается	max_execution_time = 120 в php.ini
memory_limit мал	Ошибка на больших XML	memory_limit = 256M
open_basedir ограничивает	Не может читать свои папки	Добавить путь в open_basedir
🟢 Мелкие
Проблема	Симптом	Решение
Кодировка	Кракозябры в логах	Убедиться что файлы в UTF-8
Часовой пояс	Время в логах не то	date.timezone = Asia/Novosibirsk в php.ini
5. Чек-лист перед запуском

□ PHP ≥ 7.0 — проверено
□ ext-curl — включён
□ ext-simplexml — включён
□ Файлы залиты в правильную папку
□ Права на запись (input/, json/, logs/, config/)
□ settings.json — IP 1С правильный
□ curl с сервера до 1С — работает
□ index.php открывается в браузере
□ test.php — все тесты проходят
□ Кнопка «Запустить обработку» — работает
□ Cron настроен (или пока ручной режим)
□ phpinfo.php — УДАЛЁН

Привет! Нужно разместить небольшое PHP-приложение на сервере Б24.

Что потребуется:

Папка на веб-сервере, доступная по HTTP (отдельная от Б24)
PHP 7.0+ с расширениями: curl, simplexml, json, mbstring
Права на запись для PHP-процесса в папку приложения
Сетевой доступ с этого сервера на [IP:порт 1С]
Возможность настроить cron-задачу (1 скрипт, каждые 5 минут)
Приложение не использует БД, не модифицирует Битрикс24.
Это отдельный проект в своей папке.
